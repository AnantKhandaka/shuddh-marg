PROJECT: Shuddhmarg
PURPOSE: A public directory website that lists halal-certified companies and products in India, and shows pure dharmic alternatives for every Hindu, Sikh, and Jain user. The goal is to help people make informed purchasing decisions aligned with their religious beliefs.

TAGLINE: "Jaano kya hai shuddh. Chuno jo hai dharmic."

TECH STACK:
- Frontend: Next.js 14 (App Router) + TypeScript
- Styling: Tailwind CSS
- Database: Supabase (PostgreSQL) hosted on ap-south-1 (Mumbai)
- Auth: Supabase Auth
- Data fetching: TanStack Query
- Internationalization: next-intl (English + Hindi, more later)
- Validation: Zod
- Icons: lucide-react
- Utilities: clsx, tailwind-merge
- Backend: Supabase auto REST API (Phase 1), FastAPI (Phase 2 when needed)
- Deployment: Vercel (frontend), Supabase (database + storage)

CURRENT STATUS: Step 2 (Next.js scaffold) in progress. Step 1 (Supabase schema) is complete.

SUPABASE DATABASE SCHEMA (6 tables):
---------------------------------------------

TABLE: categories
- id (serial PK)
- name (text unique)
- slug (text unique)
- type (enum: company | product | both)

TABLE: companies
- id (uuid PK)
- name (text)
- slug (text unique)
- description (text nullable)
- logo_url (text nullable)
- website_url (text nullable)
- halal_cert_body (text nullable)
- cert_number (text nullable)
- cert_document_url (text nullable)
- category (text)
- country (text default India)
- state (text nullable)
- is_verified (boolean default false)
- fts (tsvector generated, for full text search)
- created_at, updated_at (timestamptz)

TABLE: products
- id (uuid PK)
- company_id (uuid FK → companies.id cascade delete)
- name (text)
- slug (text unique)
- description (text nullable)
- image_url (text nullable)
- barcode (text unique nullable)
- category (text)
- halal_logo_visible (boolean default false)
- is_verified (boolean default false)
- fts (tsvector generated, for full text search)
- created_at, updated_at (timestamptz)

TABLE: alternatives
- id (uuid PK)
- product_id (uuid FK → products.id set null)
- company_id (uuid FK → companies.id set null)
- name (text)
- brand (text nullable)
- image_url (text nullable)
- buy_link (text nullable)
- description (text nullable)
- dharmic_type (text[] default {hindu,sikh,jain})
- is_vegetarian (boolean default true)
- is_jhatka (boolean default false)
- is_vegan (boolean default false)
- created_at, updated_at (timestamptz)

TABLE: stores (for map feature Phase 2)
- id (uuid PK)
- name (text)
- address (text nullable)
- city (text nullable)
- state (text nullable)
- lat (numeric 10,7 nullable)
- lng (numeric 10,7 nullable)
- has_dharmic_section (boolean default false)
- google_maps_url (text nullable)
- is_verified (boolean default false)
- submitted_by (uuid FK → auth.users nullable)
- created_at, updated_at (timestamptz)

TABLE: submissions (community moderation queue)
- id (uuid PK)
- type (enum: company | product | store | alternative)
- payload (jsonb)
- status (enum: pending | approved | rejected, default pending)
- submitted_by (uuid FK → auth.users nullable)
- reviewed_by (uuid FK → auth.users nullable)
- review_note (text nullable)
- created_at, updated_at (timestamptz)

RLS POLICIES:
- Public can SELECT from companies WHERE is_verified = true
- Public can SELECT from products WHERE is_verified = true
- Public can SELECT from alternatives (all rows)
- Public can SELECT from stores WHERE is_verified = true
- Public can SELECT from categories (all rows)
- Public can INSERT into submissions
- Admin uses service_role key server-side which bypasses RLS

INDEXES:
- idx_companies_slug, idx_products_slug (slug lookups)
- idx_products_company (FK join)
- idx_alternatives_product, idx_alternatives_company (FK joins)
- idx_companies_category, idx_products_category (filtering)
- idx_stores_lat_lng (map bounding box queries)
- idx_submissions_status (moderation queue)
- idx_companies_fts, idx_products_fts (full text search GIN)

TRIGGERS:
- update_updated_at() trigger on all 5 main tables (auto updates updated_at on every UPDATE)

---------------------------------------------
KEY FILES CONTENT:
---------------------------------------------

lib/supabase.ts:
- exports supabase (anon client, used in frontend)
- exports supabaseAdmin (service_role client, used server-side only)
- both typed with Database type from types/database.ts

lib/queries.ts functions:
- getAllCompanies() → all verified companies ordered by name
- getCompanyBySlug(slug) → single company
- getCompaniesByCategory(category) → filtered companies
- getAllProducts() → all verified products with company join
- getProductBySlug(slug) → single product with company join
- getProductsByCompany(companyId) → products for one company
- getAlternativesByProduct(productId) → alternatives for one product
- searchCompanies(query) → full text search
- searchProducts(query) → full text search
- getAllCategories() → all categories

lib/utils.ts:
- cn(...inputs) → clsx + tailwind-merge utility
- slugify(text) → converts text to URL slug

middleware.ts:
- next-intl locale middleware
- handles en/hi routing
- matcher excludes /admin, /api, /_next, static files
